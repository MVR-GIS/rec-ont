7  Recreation Vocabulary Review

This page is designed to support review of the controlled vocabulary that informs the recreation ontology.

It focuses on:

  • vocabulary concepts rather than ontology classes
  • broader/narrower concept structure
  • connections from vocabulary concepts to ontology resources
  • review clarity over exhaustive RDF inspection

7.1 How to use this page

Recommended review sequence:

  1. Start with the Vocabulary In Context diagram
  2. Review the Major Vocabulary Families
  3. Use the Interactive Vocabulary Explorer to inspect specific concepts
  4. Consult the Concept Inventory for a tabular audit view

As you review, consider:

  • Are these items really best modeled as concepts rather than classes?
  • Are broader/narrower relationships intuitive?
  • Are any concepts too implementation-specific for a shared vocabulary?
  • Are concept-to-ontology links clear and appropriate?

7.5 Review Notes

Interpret the graph with the following distinctions in mind:

  • Concept Scheme nodes represent organizing schemes or collections
  • Top Concept nodes usually act as major entry points in the vocabulary
  • Grouped concept nodes are heuristically clustered for reviewer readability
  • Ontology Link Target nodes are not vocabulary concepts; they are linked resources from the ontology or related model artifacts

This page is intentionally optimized for review clarity rather than full RDF completeness.

7.8 Assumptions and updates

This page assumes the collaborator-supplied vocabulary file is Turtle and primarily uses SKOS-style constructs.

If collaborators update the vocabulary regularly, the page should continue to work as long as the file remains RDF/Turtle and still exposes at least some of:

  • skos:Concept
  • skos:ConceptScheme
  • skos:prefLabel
  • skos:broader
  • skos:inScheme

If your collaborator file uses different predicates or naming conventions, the constants section near the top is the main place to adapt it.
